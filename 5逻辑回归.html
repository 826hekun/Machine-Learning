# Attrition ~ MonthlyIncome: 指定了要拟合的模型的公式，其中 Attrition 是反应变量（因变量），MonthlyIncome 是解释变量（自变量）。
# data = churn_train: 指定了数据集。
# method = "glm": 指定了使用的建模方法，这里是普通线性回归。

# 这段代码使用caret包中的train函数，通过10折交叉验证（10-fold cross-validation）来训练一个基于逻辑回归（glm）的分类模型，并且指定了两个自变量，分别为MonthlyIncome和OverTime，因此生成的模型为Attrition ~ MonthlyIncome + OverTime。
# 
# 具体来说，train函数通过交叉验证来评估模型的性能，将训练数据集分为10个相同大小的子集，每次将其中9个子集用于模型的训练，剩下的1个子集用于模型的测试。交叉验证可以避免模型对特定数据集的过度拟合，同时也可以提高模型的泛化性能。在每次训练完成后，train函数会返回一个经过交叉验证评估的模型，并计算出该模型的性能评价指标（默认为准确率）。
# 
# 在这段代码中，train函数的参数如下：
# 
# Attrition ~ MonthlyIncome + OverTime：指定了因变量（Attrition）和两个自变量（MonthlyIncome和OverTime），表示要训练的模型为基于这三个变量的逻辑回归模型。
# data = churn_train：指定训练数据集为churn_train。
# method = "glm"：指定使用逻辑回归作为分类方法。
# family = "binomial"：指定模型为二元逻辑回归模型，表示因变量为二分类变量。
# trControl = trainControl(method = "cv", number = 10)：指定交叉验证的方法为10折交叉验证（method = "cv"，number = 10），表示将数据集划分为10个子集进行交叉验证。

#install.packages("ROCR")
library(ROCR)
## Warning: 程辑包'ROCR'是用R版本4.2.3 来建造的
# Compute predicted probabilities
# 表示用 cv_model1 对 churn_train 进行预测，并且获取预测结果中“是”的概率值。
m1_prob <- predict(cv_model1, churn_train, type = "prob")$Yes
m3_prob <- predict(cv_model3, churn_train, type = "prob")$Yes

# Compute AUC metrics for cv_model1 and cv_model3
# 这段代码使用了 prediction() 和 performance() 函数，这两个函数都是来自 ROCR 包的。
# 
# 首先，prediction() 函数将模型的预测概率 m1_prob 和 m3_prob 与实际的离职情况 churn_train$Attrition 传递给 ROCR 包。通过 measure = "tpr" 和 x.measure = "fpr" 参数， performance() 函数计算了真正率 (True Positive Rate) 和假正率 (False Positive Rate)。这两个指标是ROC曲线绘制过程中的基本指标。真正率是真正类别的样本被正确地预测为正例的比率，而假正率是负类别的样本被错误地预测为正例的比率。
# 
# 最终，perf1 和 perf2 存储了不同模型的真正率和假正率，可用于后续绘制 ROC 曲线和计算 AUC（Area Under the Curve）。
perf1 <- prediction(m1_prob, churn_train$Attrition) %>%
  performance(measure = "tpr", x.measure = "fpr")
perf2 <- prediction(m3_prob, churn_train$Attrition) %>%
  performance(measure = "tpr", x.measure = "fpr")
